### --amentオプションとは

- 直前のコミットの修理

    1. 直前のコミットメッセージを変更する
    2. 直前のコミット内容を変更する

- 注意点

    1. コミットの修正をしたら、そのコミットIDは変わる
    2. <font color=red>push済みのコミットは修正してはいけない</font>  
        *ローカルのコミットIDとリモートのコミットIDが異なってしまうため

---

### コミットメッセージを変更する

```bash
git commit --ammend (-m "コミットメッセージ")
```

例
```bash
# filesのタイポ
git commit -m "add new filea"

# コミットメッセージの修正
git commit --ammend -m "add new files"
```

---

### コミット内容の変更を行う

- いらないファイルをコミットしてしまった

```bash
# いらないファイルの削除 & ステージング
git rm <対象ファイル or ディレクトリ>

# 直前コミットに含める
git commit --amend (-m "コミットメッセージ")
```

<br>

- 必要なファイルをコミットし忘れた
```bash
# 必要なファイルをステージング
git add <対象ファイル or ディレクトリ>

# 直前コミットに含める
git commit --amend (-m "コミットメッセージ")
```

<br>

- ファイルの中身の変更が不十分だった
```bash
# 対象ファイルの修正
vi <対象ファイル> #IDEで修正とか

# 必要なファイルをステージング
git add <対象ファイル or ディレクトリ>

# 直前コミットに含める
git commit --amend (-m "コミットメッセージ")
```

---

### コミットメッセージを変更しない場合

```bash
git commit --amend --no-edit
```
